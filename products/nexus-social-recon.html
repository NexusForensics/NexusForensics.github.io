Compliance Configuration

Visible-capture workflows for ToS-conscious evidence preservation.

Nexus Social Recon Compliance is being configured around a policy-gated visible-capture mode. Low-risk tools such as screenshots, visible screen recording, local OCR, manual navigation, hashing, and local reports remain available, while workflows that scrape platform DOM data, collect hidden structured data, download media streams, or bulk-capture relationship lists are disabled in compliance mode.

Preserve what is visible

Selected-region captures and visible recordings document the evidence as it appeared to the authorized user in the browser view.

Local evidence processing

OCR, reports, manifests, hashes, and case indexes are generated from already preserved local evidence artifacts.

Investigator-guided automation

Use automation where appropriate while keeping visible captures, OCR results, and evidence review clear and under investigator control.

Latest Upgrades

Local indexing now starts earlier in the workflow.

The newest Social Recon workflow separates same-computer local intelligence from shared network intelligence, so investigators can use local cross-case memory at the Intelligence level while reserving shared network lookup for Team deployments.

Local Index

Investigator cases can index OCR text, User Index identifiers, captured friend names, media signatures, targets, faces, objects, and case context on the local computer.

Local Cross-Case Lookup

Compare local cases for text, User Index links, friend identifiers, OCR, comments, captions, faces, thumbnails, objects, and visual signatures.

Global Targets

Promote faces, logos, and distinctive objects into a local target index for same-computer future-case recognition and explainable hit reports.

Shared Network Index

Team deployments extend detection beyond one computer through a governed shared index for authorized investigators.

Investigation Workflow

Capture first, then enrich without losing source context.

The workflow follows how investigators actually move through social evidence: preserve the page, mark what matters, index local signals, compare across unlocked cases when allowed, and package the record.

Step 1Open a case

Start from a local case workspace with agency and investigator context.

Step 2Capture sources

Save visible public pages, media, comments, and source-linked records.

Step 3Bookmark findings

Keep the strongest evidence visible across platform modules.

Step 4Index and compare

OCR, User Index links, friend identifiers, target checks, visual matching, and local cross-case lookup.

Step 5Export package

Generate reports, media folders, hashes, and disclosure-ready files.

Previous Capture Detection

Know when evidence has already been captured.

Nexus Social Recon alerts the investigator when a profile, post, or exact source URL has already been preserved in an available case. It can also place the current capture beside prior case evidence for direct review. This helps prevent duplicate collection, directs investigators to existing evidence, and frees more time for new findings and deeper case analysis.

Previous Capture Detection shows where an exact URL or account identity was preserved and provides direct access to the saved evidence, source URL, and case folder.

Image Case Preview places the current capture beside prior case evidence, with the matched identifier, source URL, and local evidence path available for review.
